Fault-injection benchmark for LLM agents

Most benchmarks test agents on a good day. FaultLine tests what happens on a bad one.

A fault-injection benchmark for LLM agent loops. We break a tool mid-task — a timeout, an error, malformed output — then score whether the agent recovers, degrades, spirals, or doom-loops. Every verdict is reproducible from the trace, not an LLM's opinion.

How it works

Every faulted run is compared against a fault-free baseline of the same task, then classified from telemetry alone.

01

Inject a fault

At a chosen step, the tool wrapper forces one call to fail — without touching the agent framework's internals.

hard_errortransient malformedempty timeout
02

Run the loop through it

The agent keeps going. Each fault runs two ways — surfaced to the model as a tool result, or raised straight through the loop.

caughtpropagated
03

Score the trace

Step count, repeated tool-call signatures, token cost and termination decide one verdict. Reproducible — no LLM judge.
